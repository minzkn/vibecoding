OpenCode는 터미널 기반 AI 코딩 에이전트입니다. VS Code 확장이 아닌 CLI 도구로, Claude CLI와 유사한 구조를 가지고 있습니다. 다양한 LLM을 지원하며 Go 언어로 작성되었습니다.

OpenCode에서 마이그레이션

OpenCode가 아카이브되었으므로, 다음 활성 도구로 마이그레이션을 권장합니다. 각 도구의 특성에 맞춰 선택하세요.

Continue.dev로 전환 (가장 권장)

✅ 권장 이유
  • 오픈소스: Apache 2.0 라이선스, 활발한 개발
  • 다중 LLM: OpenCode처럼 다양한 모델 지원
  • VS Code 네이티브: 별도 터미널 없이 에디터에 통합
  • Ollama 지원: 완전 무료 로컬 사용 가능
  • 풍부한 기능: @codebase, @docs 등 고급 기능 제공

마이그레이션 단계

# 1. Continue 확장 설치
VS Code Extensions → "Continue" 검색 → 설치

# 2. Ollama 연동 (OpenCode에서 Ollama 사용했다면)
Continue 사이드바 → Settings → Ollama 선택
모델: qwen2.5-coder (OpenCode와 동일)

# 3. 기존 API 키 재사용
Continue 설정에 Claude/OpenAI API 키 입력

# 4. 단축키 학습
Cmd/Ctrl + L  # 채팅 열기
Cmd/Ctrl + I  # 인라인 편집

Continue.dev 완전 가이드 →

Claude CLI로 전환 (터미널 선호 시)

ℹ️ Claude CLI 특징
  • 공식 도구: Anthropic 공식 지원, 안정적
  • 터미널 기반: OpenCode처럼 CLI 인터페이스
  • 최신 모델: Claude 4.6 Opus, 4.5 Sonnet 즉시 지원
  • 제한사항: Claude 모델만 지원 (다중 LLM 불가)

마이그레이션 단계

# 1. Claude CLI 설치
npm install -g @anthropic-ai/claude-cli

# 2. API 키 설정
export ANTHROPIC_API_KEY="sk-ant-api03-xxx"

# 3. 실행 (OpenCode 대신)
claude  # opencode 대신 claude 명령어 사용

# 4. 프로젝트 컨텍스트 설정
mkdir .claude
# .claude/instructions.md에 프로젝트 규칙 작성

Cline으로 전환 (자율 에이전트 원할 때)

ℹ️ Cline 특징
  • 자율 에이전트: 복잡한 작업을 단계별로 자동 실행
  • 터미널 실행: npm install, git commit 자동 처리
  • 브라우저 제어: 웹 테스트 자동화 가능
  • 승인 시스템: 위험한 작업은 사용자 확인

"전체 프로젝트 구조 생성해줘"

LLM 제공자별 설정

OpenCode는 settings.json에서 모든 설정을 관리합니다. Cmd/Ctrl + Shift + P → "Preferences: Open Settings (JSON)" 선택.

Claude API (Anthropic)

// settings.json
{
  "opencode.provider": "anthropic",
  "opencode.apiKey": "sk-ant-api03-xxx",
  "opencode.model": "claude-",
  "opencode.maxTokens": 8000,
  "opencode.temperature": 0.7
}

// 사용 가능한 모델:
// - claude- (최고 품질, 비싼)
// - claude- (균형, 권장)
// - claude- (빠르고 저렴)

OpenAI (GPT-4, GPT-4o)

// settings.json
{
  "opencode.provider": "openai",
  "opencode.apiKey": "sk-proj-xxx",
  "opencode.model": "gpt-4o",
  "opencode.maxTokens": 4000
}

// 사용 가능한 모델:
// - gpt-4o (최신, 빠름)
// - gpt-4-turbo (고품질)
// - gpt-3.5-turbo (저렴, 빠름)

Google Gemini

// settings.json
{
  "opencode.provider": "google",
  "opencode.apiKey": "AIzaSy...",
  "opencode.model": "gemini-2.0-flash-exp"
}

// 사용 가능한 모델:
// - gemini-2.0-flash-exp (실험, 빠름)
// - gemini-1.5-pro (고품질)

Ollama (로컬, 무료)

// 1. Ollama 설치 및 모델 다운로드
ollama pull llama3.2
ollama pull qwen2.5-coder

// 2. settings.json
{
  "opencode.provider": "ollama",
  "opencode.model": "llama3.2",
  "opencode.ollamaUrl": "http://localhost:11434",
  "opencode.maxTokens": 2000
}

// 코딩 전용 모델 (권장):
// - qwen2.5-coder:7b (최고 코딩 성능)
// - deepseek-coder-v2 (대안)
// - codellama:13b (Meta 공식)

다중 모델 설정 (고급)

// 작업별로 다른 모델 사용
{
  "opencode.provider": "anthropic",
  "opencode.apiKey": "sk-ant-api03-xxx",

  // 인라인 편집: 빠른 모델
  "opencode.inlineModel": "claude-",

  // 채팅: 고품질 모델
  "opencode.chatModel": "claude-",

  // 자동완성: 초고속 모델
  "opencode.autocompleteModel": "claude-"
}
💡 권장 설정
  • 최고 품질: Claude Sonnet (균형) (균형있는 성능)
  • 완전 무료: Ollama + Qwen2.5-Coder (로컬)
  • 빠른 속도: Claude Haiku (경량) 또는 GPT-3.5
  • 비용 최적화: 간단한 작업은 Haiku, 복잡한 작업은 Sonnet

환경 변수로 API 키 관리

보안을 위해 API 키를 settings.json에 직접 저장하지 말고 환경 변수를 사용하세요. 

# ~/.bashrc 또는 ~/.zshrc
export ANTHROPIC_API_KEY="sk-ant-api03-xxx"
export OPENAI_API_KEY="sk-proj-xxx"

# settings.json (API 키 제거)
{
  "opencode.provider": "anthropic",
  "opencode.model": "claude-"
  // apiKey 필드 없음 → 환경 변수에서 자동 로드
}
⚠️ 보안 주의

API 키를 Git에 커밋하지 마세요! settings.json을 .gitignore에 추가하거나 환경 변수를 사용하세요.

인라인 편집 (Cmd/Ctrl+K)

가장 자주 사용하는 핵심 기능입니다. 코드를 선택하고 즉시 AI에게 수정을 요청할 수 있습니다.

기본 워크플로우

  1. 편집하고 싶은 코드 블록 선택 (선택 없이도 가능)
  2. Cmd+K (Mac) 또는 Ctrl+K (Windows)
  3. 원하는 변경사항을 자연어로 입력
  4. Enter → AI가 코드 수정/생성
  5. 변경사항 확인 후 Accept 또는 Reject

Python - FastAPI REST API

# Cmd+K → "사용자 CRUD API 엔드포인트 작성. Pydantic 모델 포함"

from fastapi import FastAPI, HTTPException
from pydantic import BaseModel
from typing import List, Optional

app = FastAPI()

class User(BaseModel):
    id: int
    name: str
    email: str
    active: bool = True

users_db: List[User] = []

@app.post("/users", response_model=User)
async def create_user(user: User):
    users_db.append(user)
    return user

@app.get("/users/{user_id}", response_model=User)
async def get_user(user_id: int):
    user = next((u for u in users_db if u.id == user_id), None)
    if not user:
        raise HTTPException(status_code=404, detail="User not found")
    return user

React + TypeScript - 커스텀 훅

// Cmd+K → "데이터 페칭을 위한 커스텀 훅 작성. 로딩/에러 상태 포함"

import { useState, useEffect } from 'react';

interface UseFetchResult<T> {
  data: T | null;
  loading: boolean;
  error: Error | null;
  refetch: () => void;
}

export function useFetch<T>(url: string): UseFetchResult<T> {
  const [data, setData] = useState<T | null>(null);
  const [loading, setLoading] = useState(true);
  const [error, setError] = useState<Error | null>(null);

  const fetchData = async () => {
    try {
      setLoading(true);
      const response = await fetch(url);
      if (!response.ok) throw new Error('Network error');
      const json = await response.json();
      setData(json);
      setError(null);
    } catch (err) {
      setError(err as Error);
    } finally {
      setLoading(false);
    }
  };

  useEffect(() => {
    fetchData();
  }, [url]);

  return { data, loading, error, refetch: fetchData };
}

// Cmd+K → "gRPC 서버 기본 구조 작성. Health check 포함"

package main

import (
    "context"
    "log"
    "net"

    "google.golang.org/grpc"
    "google.golang.org/grpc/health"
    "google.golang.org/grpc/health/grpc_health_v1"
)

type server struct {
    UnimplementedUserServiceServer
}

func (s *server) GetUser(ctx context.Context, req *GetUserRequest) (*UserResponse, error) {
    // 구현
    return &UserResponse{
        Id:    req.Id,
        Name:  "John Doe",
        Email: "john@example.com",
    }, nil
}

func main() {
    lis, err := net.Listen("tcp", ":50051")
    if err != nil {
        log.Fatalf("failed to listen: %v", err)
    }

    grpcServer := grpc.NewServer()
    RegisterUserServiceServer(grpcServer, &server{})

    // Health check
    healthServer := health.NewServer()
    grpc_health_v1.RegisterHealthServer(grpcServer, healthServer)

    log.Printf("gRPC server listening on :50051")
    if err := grpcServer.Serve(lis); err != nil {
        log.Fatalf("failed to serve: %v", err)
    }
}

SQL - 복잡한 쿼리 최적화

-- 기존 쿼리 선택 후 Cmd+K → "인덱스 활용하도록 최적화하고 설명 주석 추가"

-- 최적화된 사용자별 주문 통계 쿼리
-- 인덱스: users(id), orders(user_id, created_at), order_items(order_id)
SELECT
    u.id,
    u.name,
    COUNT(DISTINCT o.id) AS total_orders,
    SUM(oi.quantity * oi.price) AS total_revenue,
    AVG(oi.quantity * oi.price) AS avg_order_value
FROM users u
INNER JOIN orders o ON u.id = o.user_id
INNER JOIN order_items oi ON o.id = oi.order_id
WHERE o.created_at >= DATE_SUB(NOW(), INTERVAL 30 DAY)
    AND o.status = 'completed'
GROUP BY u.id, u.name
HAVING total_orders > 5
ORDER BY total_revenue DESC
LIMIT 100;

응답이 느려요

  • 네트워크 속도 문제
  • 모델이 오버로드 상태

해결:

// settings.json
{
  "opencode.contextWindow": 2000,        // 기본 4000에서 축소
  "opencode.includeOpenFiles": false,    // 현재 파일만
  "opencode.model": "claude-"  // 빠른 모델
}

API 할당량 초과

에러: 429 Too Many Requests

해결:

  • Ollama로 전환 (무료, 무제한)
  • API 키 여러 개 로테이션
  • 자동완성 비활성화
// Ollama로 전환
{
  "opencode.provider": "ollama",
  "opencode.model": "qwen2.5-coder"
}

AI가 컨텍스트를 이해 못 해요

해결:

  • 관련 파일을 모두 열어두기
  • 채팅에서 명시적으로 파일 경로 언급
  • opencode.includeOpenFiles: true 설정

자동완성이 작동 안 해요

확인 사항:

// settings.json
{
  "opencode.autocompleteEnabled": true,
  "opencode.autocompleteDelay": 500  // ms, 너무 짧으면 작동 안 함
}

// API 키가 설정되었는지 확인
// Ollama의 경우 서버 실행 상태 확인
ollama list  # 모델 목록 확인

에러 메시지: "Invalid API Key"

해결:

# 1. API 키 확인
echo $ANTHROPIC_API_KEY  # 환경 변수

# 2. settings.json에서 직접 설정
{
  "opencode.apiKey": "sk-ant-api03-xxx"
}

자주 묻는 질문

Q: OpenCode vs Continue, 어떤 게 더 좋나요?

A: OpenCode는 더 가볍고 빠르며 설정이 간단합니다. Continue는 더 많은 기능(@codebase, @docs 등)을 제공합니다. 초보자나 빠른 응답이 필요하면 OpenCode, 고급 기능이 필요하면 Continue를 추천합니다.

Q: 완전 무료로 사용할 수 있나요?

A: 네! Ollama를 설치하고 OpenCode와 연동하면 완전 무료입니다. Qwen2.5-Coder 모델로 상용 LLM에 근접한 성능을 얻을 수 있습니다. Ollama 연동 가이드 →

Q: 여러 LLM을 동시에 사용할 수 있나요?

A: 네, 작업별로 다른 모델을 설정할 수 있습니다: 인라인 편집은 빠른 모델, 채팅은 고품질 모델로 분리 가능합니다.

Q: 회사 코드를 사용해도 안전한가요?

A: Ollama를 사용하면 모든 데이터가 로컬에서만 처리되어 완전히 안전합니다. 상용 API를 사용하는 경우 각 제공자의 데이터 정책을 확인하세요. (Claude는 API 데이터를 학습에 사용하지 않습니다)

Q: 성능을 더 향상시킬 수 있나요?

A: 컨텍스트 윈도우를 줄이고, 빠른 모델을 사용하고, 자동완성 지연 시간을 늘리면 체감 속도가 향상됩니다.
