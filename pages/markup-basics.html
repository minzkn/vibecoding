마크업 기초와 Markdown 실전

마크업의 본질, Markdown 문법, 문서 구조화 흐름, 실전 체크리스트를 한 번에 정리합니다.

개요

마크업은 문서에 의미를 부여하는 방법입니다. Markdown은 가독성과 속도에 특화된 경량 마크업으로, 기술 문서와 협업 문서에서 가장 널리 쓰입니다.

Markdown → 렌더러 → HTML 출력 흐름

마크업의 핵심 개념

마크업은 시각적 꾸밈이 아니라 구조와 의미를 표현합니다. 문서를 구조화하면 검색, 요약, 자동화에 유리합니다.

  • 의미 중심 구조: 제목, 단락, 목록, 코드
  • 일관성: 문서 간 동일한 규칙 유지
  • 변환 가능성: Markdown, HTML, PDF 등 다양한 출력
포인트: 문서 구조가 명확할수록 LLM이 요약, 추출, 변환 작업을 더 정확히 수행합니다.

Markdown 핵심 문법

실무에서 가장 많이 쓰는 요소만 정리합니다. 문법을 최소화하고, 팀 전체가 동일한 규칙을 적용하는 것이 핵심입니다.

  • 제목: `#` ~ `###`까지 단계적으로 사용
  • 목록: `-` 또는 `1.`로 일관성 유지
  • 강조: `**굵게**`, `_기울임_`
  • 코드: `` `inline code` `` 및 ``` fenced code ```
  • 링크: `[텍스트](URL)` 형식
  • 표: 파이프 `|`로 열 정렬

문서 구조화 흐름

문서를 빠르게 완성하려면 구조를 먼저 고정하고 내용을 채우는 방식이 효율적입니다.

  1. 문서 목표와 독자 정의
  2. 상위 섹션(h2) 3~6개 확정
  3. 각 섹션에 요약 문장 1개 먼저 작성
  4. 세부 목록과 예제로 채우기
실전 팁: h2 제목을 먼저 완성하면 TOC가 자동으로 만들어져 진행률이 명확해집니다.

LLM과의 협업 흐름

LLM에게 먼저 개요를 만들게 한 뒤 사람이 구조를 확정하면 품질과 속도를 동시에 확보할 수 있습니다.

  • 요구사항 요약 → 문서 목적/대상 확정
  • LLM으로 초안 목차 생성
  • 사람이 흐름과 용어 정리
  • LLM으로 예제와 체크리스트 확장

사람과 LLM의 역할 분담 흐름

실전 예제

Markdown을 활용해 API 요청 내용을 구조화한 예시입니다.

import anthropic

client = anthropic.Anthropic(
    api_key="$CLAUDE_API_KEY"
)

message = client.messages.create(
    model="claude-sonnet-4-5-20250929",
    max_tokens=800,
    messages=[
        {"role": "user", "content": "# 작업 요약\n- 목표: 문서 구조 확정\n- 산출물: 목차 포함 Markdown"}
    ]
)
print(message.content)
주의: Markdown 안에 너무 긴 문장을 넣으면 요약 품질이 급격히 떨어집니다.

작성 체크리스트

  • h2가 최소 3개 이상이며 중복 ID가 없는가
  • 목록과 표의 스타일이 문서 전체에서 일관적인가
  • 핵심 메시지가 첫 단락에 요약되어 있는가
  • 코드 블록에 언어 정보와 강조 클래스가 포함되어 있는가
  • 내부 링크가 최신 페이지를 가리키는가
