Markdown 스타일 가이드와 문서 품질

팀 문서를 빠르게 읽히게 하고 유지보수를 쉽게 만드는 Markdown 스타일 규칙을 정리합니다.

개요

좋은 Markdown은 문서의 신뢰도를 높이고, 리뷰 시간을 줄이며, LLM의 요약 품질도 향상시킵니다. 스타일 가이드는 문서 품질을 자동화하는 최소 단위입니다.

핵심 목표: 읽기 쉽고, 수정하기 쉽고, 자동화하기 쉬운 문서.

문서 품질이 유지되는 간단한 운영 루프

가독성 규칙

  • 한 문단은 3~5줄로 제한
  • 핵심 메시지는 첫 문장에 배치
  • 동일한 문서에서 제목 수준을 건너뛰지 않기
  • 리스트 문장 끝의 종결 표현 통일
  • 긴 문장은 두 문장으로 분리

섹션 구조화

모든 문서는 예측 가능한 순서를 가져야 합니다. 독자가 원하는 정보를 찾는 데 걸리는 시간을 줄이는 것이 목표입니다.

  1. 문서 목적과 대상
  2. 핵심 개념 정의
  3. 단계별 절차
  4. 예제 및 FAQ
추천: h2 제목은 명사형으로 통일하면 검색 결과가 더 안정적입니다.

일관성 규칙

  • 동일한 용어는 항상 같은 표기로 사용
  • 코드/명령어는 항상 인라인 코드로 강조
  • 문서 전체에서 리스트 기호를 통일
  • 표 컬럼은 3~4개로 제한
주의: 문서 내 용어가 흔들리면 검색 인덱스 품질이 급격히 떨어집니다.

코드 블록과 예제 규칙

코드 블록은 문서의 신뢰도를 결정합니다. 실제 실행 가능한 형태를 유지하세요.

import anthropic

client = anthropic.Anthropic(
    api_key="$CLAUDE_API_KEY"
)

message = client.messages.create(
    model="claude-sonnet-4-5-20250929",
    max_tokens=600,
    messages=[
        {"role": "user", "content": "문서 스타일 체크리스트를 만들어줘"}
    ]
)
print(message.content)
  • 코드 블록에 항상 언어 타입을 지정
  • 환경 변수, 비밀 키는 플레이스홀더 사용
  • 실행 결과가 중요하면 요약 문장 추가
상황 좋은 예 나쁜 예
환경 변수 `$CLAUDE_API_KEY` 실제 키 문자열
명령어 `rg -n \"query\"` 긴 설명만 있는 문장
결과 요약 + 핵심 로그 전부 붙여넣기

표와 링크 규칙

요소 권장 방식 피해야 할 것
표 3열 이내로 요약 긴 문장으로 가득 찬 표
링크 의미 있는 텍스트 "여기" 같은 모호한 표현
이미지 캡션과 설명 추가 의미 없는 장식 이미지

문서 품질 체크리스트

  • 제목만 읽어도 흐름이 이해되는가
  • 섹션 간 중복이 제거되었는가
  • 코드 블록은 실행 가능한 형태인가
  • 링크가 최신이며 명확한 목적을 갖는가
  • 문서 길이에 비해 요약이 충분한가
  • 이미지가 있다면 캡션과 의미가 명확한가

간단한 템플릿

초안을 빠르게 만들 때 사용할 수 있는 기본 템플릿입니다.

# 문서 제목

## 목적
- 이 문서가 해결하는 문제

## 절차
1. 단계 1
2. 단계 2

## 예제
```bash
command --flag value
```
