AWS Vibe Coding 가이드

AWS로 Vibe Coding을 활용하는 방법을 배웁니다. 아마존 웹서비스를 AI로 구현해보세요.

개요

AWS(Amazon Web Services)는 세계 최대의 클라우드 컴퓨팅 플랫폼입니다. 컴퓨팅, 스토리지, 데이터베이스, ML 등 200개 이상의 서비스를 제공합니다.

📋 주요 AWS 서비스
  • EC2: 가상 서버
  • S3: 객체 스토리지
  • Lambda: 서버리스 컴퓨팅
  • RDS/Aurora: 관리형 DB
  • CloudFront: CDN
  • ECS/EKS: 컨테이너 서비스
  • DynamoDB: NoSQL 데이터베이스
  • ElastiCache: 인메모리 캐시
  • SQS/SNS: 메시징 서비스
  • IAM: 자격 증명 및 액세스 관리

적합한 상황

  • 프로덕션 배포
  • 서버리스 아키텍처
  • 대용량 데이터 처리
  • ML/AI 서비스
  • 마이크로서비스
  • 글로벌 애플리케이션

AWS 리전

서비스를 배포할 리전을 선택하세요.

💡 리전 선택 가이드
  • 서울 (ap-northeast-2): 국내 사용자에게 최저 지연
  • 도쿄 (ap-northeast-1): 일본/동아시아
  • 버지니아 (us-east-1): 미국 동부, 가장 많은 서비스
  • 프랑크프루트 (eu-central-1): 유럽

스토리지

AWS 스토리지 서비스입니다. 다양한 용도에 맞는 스토리지 옵션을 제공합니다.

S3 (Simple Storage Service)

확장성 및 내구성이 높은 객체 스토리지입니다.

AWS CLI 
# 버킷 생성
aws s3 mb s3://my-bucket-name --region us-east-1

# 파일 업로드
aws s3 cp myfile.txt s3://my-bucket-name/

# 정적 웹 호스팅 설정
aws s3 website s3://my-bucket-name \
  --index-document index.html \
  --error-document error.html

# 퍼블릭 읽기 설정
aws s3api put-public-access-block \
  --bucket my-bucket-name \
  --public-access-block-configuration "BlockPublicAcls=false"

S3 스토리지 클래스

클래스 용도 비용
Standard 자주 접근하는 데이터 표준
Intelligent-Tiering 접근 패턴 변동 약간 높음
Standard-IA 잦은 접근, 장기 보관 낮음
Glacier 아카이브 매우 낮음

💡 Aurora 장점
